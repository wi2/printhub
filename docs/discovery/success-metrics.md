# Success Metrics — Slicer Profile Generator

## Measurement Philosophy

> Metrics should reflect whether users are getting real value, not just whether they are using the product.

We organize metrics into four layers:

1. **Outcome metrics** — did the user's print succeed?
2. **Product metrics** — is the product delivering the experience we designed?
3. **Growth metrics** — is the user base growing sustainably?
4. **Quality metrics** — is the profile engine getting better over time?

Each metric has a defined owner, data source, and review cadence.

---

## 1. Outcome Metrics (North Star)

These measure whether the product is delivering on its core promise.

### 1.1 First-Print Success Rate

**Definition:** Percentage of profile generation sessions where the user subsequently reports a successful first print.

| | MVP Target | 6-Month Target | 12-Month Target |
|---|---|---|---|
| First-print success rate | ≥ 75% | ≥ 80% | ≥ 85% |

**Data source:** Post-download feedback prompt ("Did your print succeed?")  
**Measurement note:** Denominator = sessions with any feedback submitted. Sessions without feedback are excluded (not assumed failed).  
**Review cadence:** Weekly  
**Owner:** Product + Engineering

---

### 1.2 Trial-and-Error Reduction (Qualitative)

**Definition:** In periodic user interviews, what percentage of users report needing 2 or fewer print attempts before a successful result (vs. their baseline before using the product)?

| | Target |
|---|---|
| Users reporting ≤ 2 attempts to success | ≥ 70% |

**Data source:** Quarterly user interviews (n=20 minimum per cohort)  
**Review cadence:** Quarterly  
**Owner:** Product

---

## 2. Product Metrics

These measure whether the product experience is working as designed.

### 2.1 Generation Completion Rate

**Definition:** Percentage of users who start filling in the form and complete a profile download.

| | MVP Target | 6-Month Target |
|---|---|---|
| Completion rate | ≥ 70% | ≥ 80% |

**Data source:** Product analytics (funnel tracking)  
**Review cadence:** Weekly  
**Owner:** Product

**Drop-off alert threshold:** If rate falls below 60%, trigger immediate UX review.

---

### 2.2 Time to Profile

**Definition:** Median time from form submission to file download being available.

| | Target |
|---|---|
| P50 generation time | < 3 seconds |
| P95 generation time | < 8 seconds |

**Data source:** Server-side timing instrumentation  
**Review cadence:** Weekly  
**Owner:** Engineering

---

### 2.3 Import Success Rate

**Definition:** Percentage of generated profiles that import into the target slicer without errors. (Measured via synthetic testing, not user-reported.)

| | Target |
|---|---|
| Import success rate | ≥ 99% |

**Data source:** Automated CI test suite — generated profiles validated against slicer import parsers  
**Review cadence:** Per deploy  
**Owner:** Engineering

---

### 2.4 Summary Comprehension Score

**Definition:** In user testing sessions, percentage of users who can correctly explain at least 3 of the 5 highlighted parameter decisions from the plain-English summary.

| | Target |
|---|---|
| Summary comprehension score | ≥ 70% |

**Data source:** Moderated usability testing (quarterly)  
**Review cadence:** Quarterly  
**Owner:** Product + Design

---

## 3. Growth Metrics

### 3.1 Weekly Active Generators (WAG)

**Definition:** Number of unique users (by session/device fingerprint in free tier) who generate at least one profile per week.

| | Month 1 | Month 3 | Month 6 | Month 12 |
|---|---|---|---|---|
| WAG target | 50 | 250 | 750 | 2,500 |

**Data source:** Product analytics  
**Review cadence:** Weekly  
**Owner:** Product + Growth

---

### 3.2 Return Generation Rate

**Definition:** Percentage of users who generate a second profile within 30 days of their first.

| | MVP Target | 6-Month Target |
|---|---|---|
| 30-day return rate | ≥ 20% | ≥ 35% |

**Data source:** Product analytics (cookie / fingerprint tracking, or account-based post-MVP)  
**Review cadence:** Monthly  
**Owner:** Product

---

### 3.3 Referral / Word-of-Mouth Rate

**Definition:** Percentage of new users who arrive via a referred link, mentioned source, or community post containing a profile generated by the tool.

| | 6-Month Target | 12-Month Target |
|---|---|---|
| Referral-sourced new users | ≥ 15% | ≥ 25% |

**Data source:** UTM tracking + attribution survey ("How did you hear about us?")  
**Review cadence:** Monthly  
**Owner:** Growth

---

## 4. Quality / Model Metrics

These measure the health of the profile generation engine over time.

### 4.1 Feedback Response Rate

**Definition:** Percentage of download sessions that receive any feedback response (Yes / No / Not yet printed).

| | Target |
|---|---|
| Feedback response rate | ≥ 30% |

**Data source:** Feedback submission events  
**Review cadence:** Weekly  
**Owner:** Product

**Why it matters:** Low response rate limits our ability to improve the engine. If below target, test friction reduction (e.g., in-browser prompt timing, email follow-up).

---

### 4.2 Failure Report Rate by Combination

**Definition:** For each printer × material × goal combination, the percentage of feedback responses that report failure.

| | Alert Threshold |
|---|---|
| Combination failure rate | > 30% triggers investigation and profile revision |

**Data source:** Feedback data, segmented by combination  
**Review cadence:** Weekly automated report  
**Owner:** Engineering + Product

---

### 4.3 Profile Coverage Index

**Definition:** Percentage of the top 100 requested printer × material × goal combinations that have a validated, production-ready profile.

| | Month 3 Target | Month 12 Target |
|---|---|---|
| Coverage index | ≥ 60% | ≥ 90% |

**Data source:** Internal combination registry + validation test results  
**Review cadence:** Monthly  
**Owner:** Engineering

## Anti-Metrics (What We Will Not Optimize For)

These are metrics we will explicitly avoid over-indexing on, as they can be gamed or lead to poor product decisions:

- **Total page views / sessions** — we care about quality of generation, not volume of browsing
- **Time on site** — longer is not better; fast success is the goal
- **Profile download count** — a download without a successful print is not a win
- **Social media followers** — vanity metric disconnected from product value
